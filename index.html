Why Iframe Approach?

Qualtrics enforces a Content Security Policy (CSP) that blocks eval() in JavaScript. WebGazer's eye tracking models require eval() to function.

By hosting experiments externally (GitHub Pages) and embedding via iframe, we bypass these restrictions while keeping everything secure and easy to use.

Benefits:

  • ✓ Actually works in Qualtrics (header-based approaches fail)
  • ✓ No hosting setup required - just copy-paste the code
  • ✓ Centrally maintained and updated
  • ✓ Easy to fork and customize if needed
